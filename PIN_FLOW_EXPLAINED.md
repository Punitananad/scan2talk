# PIN Flow Explanation - Call Masking

## 🔑 How the PIN Works

### Step-by-Step Flow:

```
1. User clicks "Initiate Call"
   ↓
2. Frontend calls: POST /gateways/call/QRCODE/
   ↓
3. Backend generates random 4-digit PIN
   Example: 4821
   ↓
4. Backend calls SparkTG API with:
   - key: 4821 (the PIN)
   - number: 9876543210 (owner's phone)
   ↓
5. SparkTG stores mapping: PIN 4821 → 9876543210
   ↓
6. Backend returns to user:
   {
     "pin": "4821",
     "call_url": "tel:01205019042,4821#"
   }
   ↓
7. User's phone dials: 01205019042,4821#
   ↓
8. SparkTG receives call with PIN 4821
   ↓
9. SparkTG looks up: PIN 4821 → 9876543210
   ↓
10. SparkTG connects call to owner's phone
```

---

## 📝 Code Breakdown

### 1. PIN Generation (Random)
```python
def generate_pin(self):
    """Generate a secure 4-digit PIN."""
    return str(random.randint(1000, 9999))
```
**Output:** `"4821"` (random 4-digit number)

---

### 2. API Call to SparkTG
```python
# Generate PIN
pin = self.generate_pin()  # e.g., "4821"

# Prepare request
auth = (self.username, self.password)  # Basic Auth
data = {
    'key': pin,                    # PIN: "4821"
    'number': owner_phone_number   # Owner: "9876543210"
}

# Call SparkTG API
response = requests.post(
    "https://telephonycloud.co.in/api/v1/mask",
    auth=auth,
    data=data,
    timeout=10
)
```

**What gets sent to SparkTG:**
```
POST https://telephonycloud.co.in/api/v1/mask
Authorization: Basic <base64(1090600:agent007)>
Content-Type: application/x-www-form-urlencoded

key=4821&number=9876543210
```

---

### 3. SparkTG Stores the Mapping
SparkTG's database now has:
```
PIN: 4821 → Phone: 9876543210
DID: 01205019042
```

---

### 4. Return Call URL
```python
call_url = f"tel:{self.DID_NUMBER},{pin}#"
# Result: "tel:01205019042,4821#"

return {
    'success': True,
    'pin': pin,                    # "4821"
    'call_url': call_url,          # "tel:01205019042,4821#"
    'expires_in_minutes': 10,
    'did_number': self.DID_NUMBER  # "01205019042"
}
```

---

### 5. User Dials
When user clicks the call URL, their phone dials:
```
01205019042,4821#
```

The `,4821#` part is DTMF tones that automatically enter the PIN.

---

### 6. SparkTG Receives Call
```
Incoming call to: 01205019042
DTMF tones received: 4821
```

SparkTG looks up PIN 4821 in their database:
```
PIN 4821 → Phone 9876543210
```

---

### 7. Call Connected
SparkTG connects the caller to `9876543210` (owner's phone).

---

## 🎯 Key Points

### PIN is Generated by Our Backend
- **NOT** hardcoded
- **NOT** from user input
- **Randomly generated** each time: `random.randint(1000, 9999)`

### PIN is Sent to SparkTG
```python
data = {
    'key': pin,              # Our generated PIN
    'number': owner_phone    # Owner's phone from database
}
```

### PIN is Embedded in Call URL
```python
call_url = f"tel:{DID_NUMBER},{pin}#"
# Example: "tel:01205019042,4821#"
```

The `{pin}` is replaced with the actual generated PIN.

## ❓ FAQ

**Q: Where does the PIN come from?**
A: It's randomly generated by our backend using `random.randint(1000, 9999)`

**Q: Who stores the PIN mapping?**
A: SparkTG stores it in their system after we send it via API

**Q: How does the phone know to enter the PIN?**
A: The `,4821#` in the tel: URL are DTMF tones that auto-enter the PIN

**Q: Can the same PIN be reused?**
A: No, each call generates a new random PIN

**Q: What if two calls get the same PIN?**
A: Very unlikely (1 in 9000 chance), but if it happens, the newer mapping overwrites the old one

**Q: How long is the PIN valid?**
A: 10 minutes, then it expires

## 📞 Summary

**PIN Flow in One Line:**
```
Generate Random PIN → Send to SparkTG → Return tel: URL with PIN → User dials → SparkTG routes call
```

**No PIN is passed from frontend!** The PIN is generated on the backend and sent to SparkTG API.
